# 工程量预估与工期排班

与 [`architecture.md`](./architecture.md) 第 8 节顺序、[`观测性平台计划.md`](./观测性平台计划.md) 里程碑一致。本文给出 **粗量级** 预估，便于排期；**非合同承诺**，实际随熟练度、范围裁剪与线上要求浮动 ±30%～50% 属正常。

---

## 1. 口径与假设

| 项 | 假设 |
|----|------|
| 人力 | **1 人**（全栈：Go + 运维 + 文档） |
| 技能 | 已会用 Docker、Prometheus/Grafana 基础；**Eino / MCP** 若首次接触，M3 段需加学习缓冲 |
| 「工程量」 | 含：**设计 + 编码 + 自测 + 文档/规则/runbook 最小更新**；不含：大规模多租户商业化、多地部署 |
| 缓冲 | 下列人日建议预留 **20%～30%** 应对联调、版本升级、环境问题 |

**两种日历换算（便于你对照自己可投入时间）**

- **全职研发**：约 **5～6 有效人日/周**（会议、杂事已扣减）。  
- **业余推进**：约 **1.5～2.5 人日/周**（每晚 + 周末一块时间）。

下文 **人日（PD）** 为「理想连续研发日」；**日历周**按全职折算（`PD ÷ 5` 并取整区间）。

## 6. 风险与砍 scope（控工期）

| 风险 | 对工期的影响 | 建议 |
|------|----------------|------|
| 先做大而全目录模型 | +2～4 周 | 先做 **只读目录 + 单环境**，再演进多租户 |
| MCP 与 platform-api 合并/拆分犹豫 | +1～2 周 | **尽快 ADR 定一种**，再实现 |
| Eino + LLM 同时上 | +2～5 周 | **规则 + 假 LLM / 离线** 先跑通评测 |
| 过早上 K8s 全套 | +4～8 周 | **Compose 验收控制面** 再迁 Helm |
| 阶段3-5 并行开发 | -3～6 周 | 需要更熟练的上下文切换 |
