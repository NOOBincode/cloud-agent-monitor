# 可观测性能力成熟度模型

本文定义可观测平台的能力成熟度标准，用于评估当前平台状态、规划演进路径、对标业界标杆。

---

## 1. 成熟度等级定义

| 等级 | 名称 | 特征 | 典型表现 |
|------|------|------|----------|
| **L1** | 基础监控 | 有数据，被动响应 | 出问题后才知道，靠人工排查 |
| **L2** | 主动观测 | 有告警，主动发现 | 告警触发响应，有基础 Runbook |
| **L3** | 智能诊断 | 有分析，辅助定位 | 自动关联分析，提供诊断建议 |
| **L4** | 预测预防 | 有预测，提前干预 | 异常预测、容量预警、自动修复 |
| **L5** | 自治运营 | 有闭环，自动治理 | 自愈、自优化、持续改进 |

| 日志采集 | 无结构 | 结构化 | 自动解析 | 智能过滤 | 实时分析 |

| 采集器管理 | 无 | 手动部署 | 配置下发 | 健康监控 | 自动扩缩 |

|--------|----|----|----|----|----|

| 能力项 | L1 | L2 | L3 | L4 | L5 |
|--------|----|----|----|----|----|
| 访问控制 | 无 | 基础认证 | RBAC | 细粒度权限 | 动态权限 |

| 审计日志 | 无 | 基础日志 | 完整审计 | 合规报告 | 自动合规 |
| 数据加密 | 无 | 传输加密 | 存储加密 | 端到端加密 | 零信任 |

### 2.8 多租户能力

| 能力项 | L1 | L2 | L3 | L4 | L5 |
|--------|----|----|----|----|----|
| 租户隔离 | 无 | 逻辑隔离 | 数据隔离 | 资源隔离 | 完全隔离 |
| 配额管理 | 无 | 手动配额 | 自动配额 | 智能配额 | 弹性配额 |
| 计费计量 | 无 | 基础计量 | 详细计量 | 成本分摊 | 自动计费 |
| 自助服务 | 无 | 基础自助 | 完整自助 | 智能自助 | 自动化运营 |

**当前状态**：L1（单租户，预留多租户）

### 2.9 观测系统开销控制能力

| 能力项 | L1 | L2 | L3 | L4 | L5 |
|--------|----|----|----|----|----|
| 采样策略 | 无采样 | 固定采样率 | 分层采样 | 自适应采样 | 智能采样 |
| 开销预算 | 无 | 手动监控 | 软限制 | 硬限制 | 自动调节 |
| 基数控制 | 无 | 手动识别 | 自动检测 | 自动限制 | 预测控制 |
| 采集优化 | 无 | 批量上报 | 本地预聚合 | 智能聚合 | 边缘计算 |

**当前状态**：L1（无开销控制机制）

**关键指标**：
- **CPU 开销**：观测系统自身 CPU 占用应 < 2%
- **内存开销**：观测系统自身内存占用应 < 512MB
- **网络开销**：观测数据传输带宽应 < 10Mbps
- **基数限制**：单指标 label values < 1000，series < 10000

### 2.10 AI Infra 观测能力

| 能力项 | L1 | L2 | L3 | L4 | L5 |
|--------|----|----|----|----|----|
| GPU 指标 | 无 | 基础指标 | DCGM 集成 | MFU 计算 | 效率优化 |
| 推理性能 | 无 | TTFT/TPOT | 阶段分解 | 瓶颈定位 | 自动优化 |
| Token 追踪 | 无 | 基础计数 | 成本归因 | 预算控制 | 智能路由 |
| 模型审计 | 无 | 调用记录 | 完整审计 | 安全检测 | 合规报告 |

**当前状态**：L1（Schema 已定义，实现中）

**MFU 关键指标**：
- **MFU (Model FLOPS Utilization)**：实际 FLOPS / 理论峰值 FLOPS
- **目标值**：训练 MFU > 50%，推理 MFU > 30%
- **告警阈值**：MFU < 20% 需排查效率问题

## 4. 演进路线图

### 4.1 短期目标（M2-M3）：L2 → L3

```
目标：从「主动观测」升级到「智能诊断」

关键能力：
├── SLO/SLA 管理        → 服务目录 L3
├── 告警路由与通知      → 告警能力 L3
├── 服务拓扑            → 可视化 L3
├── 分布式追踪          → 数据采集 L3
└── Eino 智能诊断       → AI/智能 L2
```

### 4.2 中期目标（M4-M5）：L3 → L4

```
目标：从「智能诊断」升级到「预测预防」

关键能力：
├── 异常检测            → AI/智能 L3
├── 容量预测            → AI/智能 L4
├── 长期存储与降采样    → 数据存储 L4
├── 多租户完善          → 多租户 L3
├── On-call 集成        → 告警能力 L4
├── MFU 观测            → AI Infra L3
└── 自适应采样          → 开销控制 L3
```

### 4.3 长期目标（M5+）：L4 → L5

```
目标：从「预测预防」升级到「自治运营」

关键能力：
├── 自动修复            → 告警能力 L5
├── 自愈闭环            → 服务目录 L5
├── 智能分层存储        → 数据存储 L5
├── 自动化运营          → 多租户 L5
└── 可解释 AI           → AI/智能 L5
```
